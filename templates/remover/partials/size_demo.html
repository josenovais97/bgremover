{% load static %} {% comment %} Comparison demo for tools whose output looks IDENTICAL to the input, where the only thing that changes is a number — bytes (convert, compress) or pixel dimensions (resize). The drag-to-compare slider in demo.html is wrong for these: it reveals no visible difference by design, so it reads as broken — or worse, as if the tool altered the picture. This shows the thing that actually changed instead. Params: image - static path to the sample image (one image; there is no "after") before_label - format/state of the source, e.g. "PNG" or "Original" before_size - the source's number, e.g. "2.1 MB" or "4000 × 3000" after_label - format/state of the result, e.g. "WEBP" or "Resized" after_size - the result's number, e.g. "0.3 MB" or "1200 × 900" after_pct - result as a % of the source (bar width), e.g. 14 saving - headline badge, e.g. "86% smaller" caption - small helper line under the frame aspect - Tailwind aspect class, defaults to "aspect-[4/3]" The numbers are illustrative of a typical job, not measured from `image` — keep them plausible, and keep after_pct consistent with the two values either side. {% endcomment %}
