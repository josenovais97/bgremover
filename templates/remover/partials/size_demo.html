{% load static %} {% comment %} File-size comparison demo, for tools where the output looks IDENTICAL to the input and only the bytes change (convert, compress). The drag-to-compare slider in demo.html is wrong for those: it reveals no visible difference by design, so it reads as broken — or worse, as if the tool altered the picture. This shows the one thing that actually changed instead: the size. Params: image - static path to the sample image (one image; there is no "after") before_label - format/state of the source, e.g. "PNG" before_size - human size of the source, e.g. "2.1 MB" after_label - format/state of the result, e.g. "WEBP" after_size - human size of the result, e.g. "0.3 MB" after_pct - result size as a % of the source (bar width), e.g. 14 saving - headline badge, e.g. "86% smaller" caption - small helper line under the frame aspect - Tailwind aspect class, defaults to "aspect-[4/3]" {% endcomment %}
